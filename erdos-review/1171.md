# Review: Erdos Problem 1171

**File:** `FormalConjectures/ErdosProblems/1171.lean`
**Source:** [erdosproblems.com/1171](https://www.erdosproblems.com/1171)
**Status:** OPEN

## Original Problem

For all finite $k < \omega$, does $\omega_1^2 \to (\omega_1 \cdot \omega, 3, \ldots, 3)_{k+1}^2$?

This asks whether the ordinal $\omega_1^2$ has the partition property that every $(k+1)$-coloring of pairs admits either a monochromatic copy of $\omega_1 \cdot \omega$ in the first color, or a 3-element monochromatic set in one of the other $k$ colors.

Known: Baumgartner proved, assuming Martin's axiom, that $\omega_1 \cdot \omega \to (\omega_1 \cdot \omega, 3)^2$.

## Current Formalization

```lean
theorem ramsey_omega_one_squared_multicolor :
    answer(sorry) ↔
      ∀ (k : ℕ),
      ∀ (omega_one_sq : Type u) [Infinite omega_one_sq] (omega_one_omega : Type u),
        PartitionRelation omega_one_sq 2 (Fin (k + 1))
          (fun i : Fin (k + 1) => if i.val = 0 then omega_one_omega else ULift.{u} (Fin 3)) := by
  sorry
```

## Analysis

### What the formalization captures

| Aspect | Captured? | Notes |
|--------|-----------|-------|
| Question format | Yes | `answer(sorry) ↔` is correct |
| Universal over finite k | Yes | `∀ (k : ℕ)` |
| Multi-color structure | Yes | First color gets a large target, remaining k colors get 3 |
| Coloring of pairs | Partially | Uses PartitionRelation with r=2 |
| Category/AMS tag | Yes | `@[category research open, AMS 03]` is correct |

### Critical issues

1. **omega_one_omega is universally quantified (Critical):** The formalization says `∀ ... (omega_one_omega : Type u)`, but this type is supposed to represent the specific ordinal $\omega_1 \cdot \omega$. As universally quantified, the statement demands the partition relation hold for *every* type as the first-color target, which is far too strong. It should either be existentially quantified with cardinality constraints or defined as a specific type.

2. **No cardinality constraints on ω₁²:** `omega_one_sq` only has `[Infinite omega_one_sq]`, with no constraint linking it to the ordinal $\omega_1^2$.

3. **No cardinality constraint on ω₁·ω:** Even if omega_one_omega were properly quantified, there is no constraint ensuring it has the cardinality/order type of $\omega_1 \cdot \omega$.

4. **Multi-color target structure is well-done:** The use of `fun i => if i.val = 0 then omega_one_omega else ULift.{u} (Fin 3)` is a good structural encoding of "first color gets a large set, remaining colors get 3-element sets." The `ULift` for universe compatibility is a reasonable technical choice.

### Minor issues

5. **PartitionRelation colors all finsets:** Same issue as in 1170 — the coloring domain is all finsets, not just pairs.

## Verdict

**Confidence: 15%**

The multi-color partition structure is well-encoded, but the universal quantification over omega_one_omega (instead of constraining it to $\omega_1 \cdot \omega$) and the lack of cardinality constraints on the source type make the statement far stronger than the original problem. The core combinatorial question about specific ordinals is lost.
